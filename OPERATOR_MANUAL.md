# 👑 PokeAssist 운영자(Administrator) 매뉴얼

이 문서는 PokeAssist 플랫폼의 최고 관리자이자 운영자인 대표님을 위해 작성된 통합 운영 가이드라인입니다.

---

## 🚀 1. 향후 코드 수정 및 업데이트 (CI/CD 배포망)

현재 PokeAssist의 모든 소스 코드는 대표님의 GitHub 저장소(`main` 브랜치)와 Vercel(프론트엔드), Render(백엔드API)에 완벽하게 묶여있습니다.

### 업데이트 루틴 (단 3단계)
1. **VS Code에서 수정 및 확인**
   - 로컬에서 수정하며 `pnpm dev`로 먼저 눈으로 확인합니다.
2. **GitHub로 쏘기 (VS Code 터미널)**
   ```bash
   git add .
   git commit -m "여기에 무엇을 수정했는지 메모 작성"
   git push origin main
   ```
3. **손 뗴고 기다리기 (자동 배포)**
   - 위 명령어를 치자마자 약 2분 내외로 Vercel과 Render가 업데이트를 감지하고 새로운 코드를 전 세계에 자동 배포합니다. 

**(💡 꿀팁)** 만약 백엔드(api) 관련 코드는 하나도 안 건드리고 프론트(web)의 버튼 색깔, 글씨만 바꿨다면 Vercel 혼자 조용히 30초 만에 업데이트를 끝냅니다. 반대로 백엔드 로직만 바꿨다면 Render 혼자 업데이트됩니다. 모노레포의 장점입니다!

## 🗄️ 2. 데이터베이스(Supabase) 직접 관리

유저들이 가입하거나 팀을 저장하면 모두 대표님의 **Supabase** 클라우드 컴퓨터에 저장됩니다. (로컬 Docker 폴더는 더 이상 보지 않으셔도 됩니다.)

1. **접속:** [Supabase Dashboard](https://supabase.com/) 에 로그인
2. **Table Editor 접속:** 좌측 메뉴의 '테이블 에디터(표 모양 아이콘)'를 누르면 엑셀처럼 생긴 모든 DB 표가 나타납니다.
3. **핵심 테이블 설명:**
   - `users`: 아까 만드신 계정(email, displayName 등)이 들어있습니다. 유저를 강퇴시킬 때 여기서 줄을 석제하면 됩니다.
   - `teams` / `team_slots`: 유저들이 짠 파티 엔트리가 원본 그대로 보관되어 있습니다.
   - `pokemon_names` / `move_names`: 1단계에서 들이부었던 한국어 번역 사본입니다.
4. **수정:** 엑셀처럼 칸을 꾹 눌러서 글씨를 바꾼 뒤 엔터를 치면 즉각 실제 서비스에 반영됩니다.

## 🤖 3. n8n을 활용한 메타 정보 '자동 수압기' 구축 가이드 (Next Step)

포켓몬스터의 메타와 유행(Usage), 신규 규정 등은 주기적으로 달라집니다. 이 데이터를 대표님이 매달 손으로 타이핑해서 넣을 수는 없습니다. 

**n8n(자동화 툴)로 우리가 만들 미래의 파이프라인:**

1. **트리거 (Trigger):**
   - 매월 1일 새벽 4시마다 n8n이 동작하도록 세팅합니다.
2. **HTTP Request 노드:**
   - Smogon 공식 API (또는 Pokemon Showdown Stats 서버)를 찔러서 최신 랭크배틀 `.txt`나 `.json` 형태의 이번 달 유행 통계를 다운로드 받습니다.
3. **Data Parsing (정제 노드):**
   - 대표님만의 자바스크립트 코드 블럭으로 "1위 날개치는머리 42%", "2위 망나뇽 38%" 형태의 JSON으로 깔끔하게 변환시킵니다.
4. **PostgreSQL 노드 (도착지):**
   - 정제된 최신 유행 JSON 데이터를 대표님의 **Supabase DB (`pokemon_usage`, `item_usage` 테이블)** 속으로 `INSERT` 또는 `UPDATE` 시킵니다.
   - 이렇게 되면 다음날 아침 유저들이 PokeAssist <메타 분석> 탭을 눌렀을 때, 사이트 코드를 단 한 줄도 수정하지 않았는데 랭킹 표가 어제와 다르게 자동으로 최신화됩니다!

### 🎯 앞으로 우선적으로 도입할 n8n 워크플로우 3대장:
1. **Smogon 티어 & Usage 월간 자동 동기화 워크플로우**
2. **Pikalytics 기반 포켓몬별 주력 기술 & 도구 & 테라스탈 빈도 자동 갱신**
3. **새로운 한국어 번역 명칭(신작/DLC 발매 시) 자동 감지 후 DB 삽입**

(이 모든 것들은 n8n을 통해 시각적으로 블록을 찍어 누르며 만드실 수 있습니다.)

## 🚨 4. 트러블슈팅 및 장애 대처법

- **"배포(Push)를 했는데 Vercel에서 빨간 글씨 에러가 뜨고 안 바뀐다!"**
  - 원인: 프론트엔드 코드에 오타나 타입(TS) 에러가 숨어있을 확률 100%. 로컬에서 터미널을 켜서 `pnpm build --filter web` 을 쳐보고 왜 에러가 나는지 확인해서 수정 후 다시 푸시합니다.
  
- **"로그인이 또 갑자기 안 된다. (CORS 에러 뿜뿜)"**
  - 원인: 환경 변수 `CORS_ORIGIN`에 이상한 주소(슬래시 등)가 끼어있거나 렌더 서버가 잠든 콜드 스타트(Cold Start) 상태일 수 있습니다. (무료 서버는 15분 접속이 없으면 잡니다. 최초 1회 접속 시 로딩이 1분 정도 긴 것은 정상입니다.)
